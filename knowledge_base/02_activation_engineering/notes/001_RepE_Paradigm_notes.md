# 001 | Representation Engineering: 表征工程的范式宣言

**Zou et al. 2023 | arXiv:2310.01405 | NeurIPS 2024**

---

## 一、核心论点

### 1.1 范式转换：从自下而上到自上而下

Zou等人提出表征工程（Representation Engineering, RepE）的核心理念：不从单个神经元或电路层面理解模型（自下而上），而是直接在高层表征层面读取和控制模型行为（自上而下）。这一方法论转换的认识论基础是：与高层概念（如诚实、安全、权力寻求）相关的信息已经在前向传播中被编码为表征空间中的特定方向，可以被直接提取和操控。

### 1.2 三段式流水线

RepE将表征操控系统化为三个阶段：
- **表征识别（Representation Identification）**：使用对比数据集或探测技术定位编码特定概念的激活方向
- **表征操作化（Representation Operationalization）**：将原始方向向量转化为标准化、可用的steering工具
- **表征控制（Representation Control）**：在推理时通过向目标层添加经过缩放的投射来调节模型行为

### 1.3 认识论主张

RepE隐含地主张：理解模型行为可以通过操控其高层表征来实现——而不需要理解这些表征是如何从底层计算中涌现的。这是一个实用主义色彩浓厚的认识论立场。

## 二、方法论

### 2.1 方向向量的构建

使用对比刺激对（如诚实 vs 不诚实）收集模型激活，通过均值差方法构建方向向量：
```
d_concept = mean(activations_positive) - mean(activations_negative)
```

### 2.2 操控的施加

在推理时，将方向向量乘以一个缩放系数后添加到目标层的残差流：
```
a'_layer = a_layer + c * d_concept
```

### 2.3 评估维度

- 诚实性检测和增强
- 道德立场的读取与控制
- 权力寻求倾向的检测
- 跨模型和跨任务泛化

## 三、核心发现

1. **线性方向假设成立**：神经网络高层表征中存在与高层概念对应的线性方向，可通过简单的均值差方法提取
2. **操控有效性**：线性干预可以有效改变模型行为，无需重新训练或微调
3. **方法效率**：RepE比基于prompt的方法更稳健，比微调更高效
4. **领域标准化**：为表征工程建立了统一的语法和评估标准

## 四、与本研究的相关性

### 4.1 干预主义因果理论的直接体现

RepE的实验范式——通过主动操纵表征来探测其因果角色——恰好符合James Woodward的干预主义因果理论。这为本项目提供了核心分析案例：激活工程如何实现干预主义所要求的'通过操纵来理解'？

### 4.2 '理解 vs 控制'的张力

RepE的核心哲学张力在于：通过表征操控我们获得的是对模型的真正理解，还是仅仅是一种控制能力？这个问题贯穿整个激活工程领域——当我们能精确地操控一个系统的行为时，我们在什么意义上'理解'了它？

### 4.3 线性假设的认识论地位

RepE的线性方向假设——'概念（如诚实）在表征空间中对应一个方向'——在认识论上是一个根本性的选择。这一假设的哲学基础是什么？它是经验发现（被后续工作不断证实或质疑）、理论推导（如015的工作）、还是方法论便利？

## 五、开放问题

1. **线性特征的完整性**：非线性概念（如'讽刺'）能否被单个线性方向有效捕获？
2. **方向的多语义性**：均值差方法提取的方向是否可能混合了多个概念的信息（后续工作如SRA、SDCV直接回应此问题）？
3. **表征的context依赖**：同一概念在不同上下文中的表征方向是否一致——如果不一致，我们应如何理解概念的统一性？
4. **操控的副作用**：线性干预不可避免地影响其他概念方向——这些'附带损伤'的界限在哪里（COAST对此进行了形式化）？
5. **认识论闭环**：我们通过操纵表征来理解模型，但这种理解又依赖于我们对操纵效果的解读——是否存在一个认识论闭环？
